## P2-2 形態(Form)切換　[暫緩]

**目標**：切換貓名顯示形態（基礎/進化/真實/究極）。

**做法**：
- godfat `name` 參數（`0/1/2/3`，確認自 fixture `#name_select`）。
- `view.html` 加「形態」下拉；`view.js` 讀值，change 時重抓。
- `lib/godfat.js` `buildEventUrl` 增加 `name`。
- `lib/cache.js` `cacheKey` **需納入 `name`**，避免快取錯置。

**工程量**：小～中　**風險**：低（純透傳）　**效益**：中

⚠️ 地區（`lang`）切換**不做**：卡池與事件 ID 是地區相關，跨區會讓既有 event 失效。沿用開啟時的 lang。

## P2-3 Details / Add future ubers（透傳，進階）　[暫緩]

**目標**：進階使用者可看種子數值（Details）、模擬未來 uber 洗牌（ubers）。

**做法**：
- `buildEventUrl` 增加 `details`（`1`）與 `ubers`（`0..5`）。
- `view.html` 加 Details 勾選、未來 uber 下拉；change 重抓。
- `cacheKey` **納入 `details`/`ubers`**。

**工程量**：中　**風險**：中　**效益**：中低（進階向）

⚠️ `details=1` 會改變 godfat 表格結構（多出種子數值欄）。**實作前先抓 `&details=1` 一頁驗證 parser 選擇器**仍能取到 cat 格，必要時調整 `lib/parser.js`。

## P2-4 貓咪圖片＋詳情連結　[暫緩]

**目標**：(a) 貓名連 godfat 單位頁；(b) 格子顯示貓咪圖片，文字/圖片切換。

**做法**：
- 詳情連結：每格貓名連 `//bc.godfat.org/cats/<id>`；`<id>` 來自 🐾 連結 → `lib/parser.js` **加抓 cat id**（`span a[href*="/cats/"]`）。
- 圖片：抓一頁 `&display=image` 確認 `<img>` src 樣式（godfat `/asset/...`），再決定 URL 組法。
- `manifest.json`：view 頁 CSP **需加** `content_security_policy` 允許 `img-src https://bc.godfat.org`（目前無自訂 CSP，預設禁外部圖）。

**工程量**：中～大　**風險**：中高（CSP/URL 需驗證；圖片多時版面與效能）　**效益**：高（辨識度大增），但成本最高

## P2-5 保證模式的換軌落點　[暫緩，優先序最低]

**目標**：模擬保證（`force_guaranteed`）時，標出抽完保證後落在哪一格（類似已完成的重複稀有 ↪ 落點）。

**做法**：先抓 `&force_guaranteed=11` 一頁，確認保證落點是文字箭頭、還是只藏在 `G` 格 `<a href>` 的 seed/last。
- 若有可解析的落點 → 比照 `dupe` 在 `cell.guaranteed.to` 標示。
- 若只在 href → 評估是否值得。**先調查再定。**

**工程量**：未定（先調查）　**風險**：未知　**效益**：中

## 共通修改重點（任一項實作時注意）

- `lib/godfat.js`：`buildEventUrl` 擴充新參數（純函式，補單元測試）。
- `lib/cache.js`：`cacheKey` 納入任何影響結果的新參數，避免快取錯置；視需要升版 `bcsp:v3`。
- `lib/parser.js`：P2-4 加抓 cat id；P2-3 視 details 模式調整選擇器。
- 沿用既有測試框架（`node:test` ＋ fixture），新參數/解析皆補測試。
- **每項實作後先自驗（`npm test` 全綠＋瀏覽器 E2E 對照 godfat 原頁）再交付。**
