# 7. 保證抽（確定11連）與重複稀有的交互（待實測驗證）

> 緣由：比較頁顯示 1A 同時有「重複稀有 ↪ 2B」與「保證: 前田慶次 ↪ 11B」，
> 釐清這兩者是不同抽法的結果，並用 godfat 實測佐證。
> 範例情境：seed=4066970932、event 2026-06-22 13週年精選轉蛋、Last roll=人魚貓(id 150)、
> force_guaranteed=11、lang=tw。

## 結論（依 godfat 實測，待遊戲內最終驗證）

1A 這格同時帶兩種抽法的結果，要分清楚：

### ① 普通單抽/連抽（非確定）
- 1A 天然＝人魚貓，但 Last roll 也是人魚貓 → 連兩隻同名稀有被禁 → **重抽成 樹剪貓、換軌到 2B**。
- 對應 godfat：`pick('1AR')` ＝「樹剪貓 -> 2B」。

### ② 確定11連（Simulate guaranteed = 11）
- 拿到保證超激 **前田慶次**，抽完落 **11B**。
- 對應 godfat：`pick('1AG')` ＝「前田慶次 -> 11B」。

## 關鍵實測發現

用 DOM 的 `pick()` id 確認歸屬（非看版面位置）：

| onclick | 內容 | 屬於 |
|---|---|---|
| `pick('1AG')` | 前田慶次 → 11B | **1A 保證** |
| `pick('1BG')` | `<- 12A 猿帝・悟空貓` | **1B 保證**（backtrack）|
| `pick('2AG')` | 猿帝・悟空貓 → 12B | 2A 保證 |

→ **1A 保證確實是 前田慶次**；猿帝・悟空貓 是 1B（與 2A）的，不是 1A。

**確定11連的結果與「1A 撞 last 的重複稀有」無關**：
- 比對「有設 last=150（人魚貓）」與「未設 last」兩種狀態，godfat 的 `pick('1AG')` 完全一致
  （都是 前田慶次 → 11B，連算出的 seed=507120225 / last=73 都相同）。
- 代表 godfat 保證欄以「乾淨起點」計算，確定11連的結果由種子位置決定，
  不受 1A 那個（對上一抽的）重複稀有影響。

## 2026-07-03 修訂：godfat 同時建模兩種起手（先前調查漏看 RG 格）

「godfat 保證欄以乾淨起點計算」的結論**只對了一半**：`G` 格確實是乾淨起手，但 godfat
**另有 `RG` 字尾格**＝**撞名狀態起手**的保證（例：戰國武將頁 `59AG`「伊達政宗 -> 69B」
與 `59ARG`「<- 70A 真田幸村」並存）。之前只查了 `pick('1AG')`，漏了變體。

**已驗證的完整機制**（fixture `dupe-forced11-banner.html`，578 G + 13 RG 共 591 格
逐一驗證零例外，見 `test/route.gu.test.js`）：

- 確定連抽（G 抽）= G−1 次單抽（含中途重複換軌，觸發條件為**有效稀有度**是稀有——
  非祭池 `supa_fest` 也觸發）+ 保證抽（**只耗一顆種子** → 落點為最後位置滑半步：
  `(q,A)→(q,B)`、`(q,B)→(q+1,A)`）。
- 撞名起手（`RG`）＝第一抽先重抽換軌、其餘照上式。
- 同一顆種子位置決定的保證 Uber，跨所有 G/RG 變體一致。
