# 擁有清單頁：篩選、圖片、型態切換 — 設計

日期:2026-07-10。基底:`feat/owned-ui`(收藏帳本視覺)之上。

## 目標

清單頁三個增強:(1)篩選列(稀有度/擁有狀態/佔位名);(2)貓咪圖示;
(3)型態(Base~Ultra)名稱與圖片全域切換。

## 已驗證的資料來源(2026-07-10 實測)

1. **圖片**:BC 資訊網(battlecatsinfo.github.io,GitHub Pages 靜態站)
   `https://battlecatsinfo.github.io/img/u/{遊戲編號}/{型態}.png`
   - **遊戲編號 = godfat 貓 id − 1**(實測:`u/0/0`＝貓咪、`u/448/0`＝宮本武藏)。
   - 型態索引:0=基本、1=二階、2=三階(True)、3=四階(Ultra);該貓沒有的階段回 404。
   - 圖 104×79、數 KB;武藏圖檔恰有 0–2 三階,與 godfat title 的三個型態名對齊
     →「型態序＝圖檔序」成立。
2. **型態名稱**:godfat /cats 目錄 title 第一行「`*名A | 名B | 名C`」**依階段排序**,
   資料已在每日快取來源中,只需以「有序陣列」形式另存。
3. `<img>` 標籤載入外站圖片不需 host_permissions(MV3 擴充頁預設 CSP 不限 img-src)。

## 已定案方向(使用者選擇)

- 篩選:**稀有度篩選＋擁有狀態三態(全部/已擁有/未擁有)＋佔位名篩選**。
  （※實作後正名「召喚」——編號單位實為貓咪附帶的召喚屬性,非未譯名貓;見文末〈實作後變更〉。）
- 型態切換:**全域**——頂部選擇器一次切整頁名稱與圖;缺該階的貓退回最高可用階;
  hover 可看全部型態名。
- 圖片:**外連 BC 資訊網**,lazy 載入、README 註明出處;圖缺/站掛退回純文字,功能不壞。
- 模型限制(已向使用者說明):擁有清單記「貓」層級(godfat o 碼亦然),
  型態切換純顯示,不記錄練到哪一階。

## 資料模型變更

- `catlist.js` 新增 `parseCatForms(doc)` → `Map<id, string[]>`(有序型態名,
  來源同 title 第一行;佔位名貓為單元素陣列)。既有 `parseCatList` 不變。
- `catlist-loader.js` 快取 schema v2:**鍵改 `bcsp:cats2:${lang}`**(格式變更必換鍵版本
  ——上次的教訓),值 `{date, names, forms}`;順手移除舊鍵 `bcsp:cats:${lang}` 孤兒。
  `loadCatList` 回傳改為 **`{ catMap, formsById } | null`**。
  - 呼叫端同步:view.js 解構取 `catMap`(不用 forms);owned 頁直接用 `formsById`
    (刪除現行手工組 formsById 的迴圈)。
- `cache.js` `SETTINGS_KEYS` 加 `bcsp:owned-form`(全域型態偏好,重載沿用)。

## UI 設計(owned 頁)

- **工具列加「型態」選擇器**:基本/二階/三階/四階;切換時就地更新各卡片的名字與圖
  (不重繪分組,保留收合與捲動狀態);每貓實際顯示階 = min(選擇, 該貓最高階)。
  卡片 tooltip 列全部型態名(「基本名 | 二階名 | …」)。
- **卡片加圖示**:checkbox 右側 42×32 縮圖(`object-fit: cover`、`loading="lazy"`、
  底色 `--bar-track` 佔位);載入失敗(404/站掛)隱藏圖、只留名字
  (error 不冒泡 → `#groups` 以 capture 委派監聽)。網格 minmax 200→230px。
- **篩選列**(搜尋框旁,均為顯示過濾、不持久化):
  - 稀有度 chips:全部/傳說/超激/激稀有/稀有/特殊/基本(單選,隱藏不符的整個分組);
  - 擁有狀態三態下拉:全部/已擁有/未擁有(**取代**現有「只看未擁有」勾選);
  - 佔位名下拉:顯示/隱藏/只看(佔位名判定:代表名符合 `/^\d+[-_]\d+$/`)。
  - 與搜尋為 AND 關係;統計與進度列不受篩選影響(恆為全量)。

## 風險

1. **外站依賴**:BC 資訊網改網址結構或關站 → 圖全部 404 → onerror 退純文字,
   清單功能不受影響;README 註明出處與此風險。
2. **id 偏移假設**(godfat id − 1):已雙樣本驗證(id 1、449);若個別貓不符
   → 該貓圖錯或 404,不影響資料正確性(名字/稀有度/擁有仍以 godfat 為準)。
3. 型態序與圖檔序不一致的個別貓:同上,顯示層退化,不影響資料。

## 施工切分(分支 `feat/owned-gallery`,基於 feat/owned-ui)

1. `feat(lib)`:parseCatForms＋loader v2(鍵版本/回傳形狀/舊鍵清理)＋SETTINGS_KEYS＋
   view.js/owned.js 呼叫端對齊＋測試
2. `feat(owned)`:型態全域切換＋卡片圖示(lazy/onerror 退化)
3. `feat(owned)`:篩選列(稀有度 chips/擁有三態/佔位名)
4. `docs`:README(功能說明＋圖片出處 attribution＋依賴風險)

## 實作後變更(2026-07-10～11 驗收期間)

以下與上文不一致處,以本節為準:

1. **「佔位名」正名「召喚」**(a83b104):使用者對照遊戲確認,`/^\d+[-_]\d+$/` 命中的
   編號單位(817-1 等 21 隻)是**貓咪附帶的召喚屬性單位**,不是「尚無譯名的未來貓」。
   本文所有「佔位名」皆應讀作「召喚」;UI 標籤、README、程式識別字
   (`SUMMON_RE`/`#summon-state`)已全面換名,判定規則與篩選行為不變。
2. **缺圖改「往上一階找」**(c196794):「遠古的蛋」系列(24 隻)BC 資訊網僅存孵化後
   (第三型態)的圖,0/1.png 皆 404 → onerror 從「直接隱圖留文字」改為逐階遞增重試至
   該貓最高階,全缺才退純文字。副作用「蛋名配孵化圖」經使用者選項確認接受。
3. **新增 JSON 檔案備份**(2ba3bd5):使用者手動輸入全清單後要求防遺失——
   「匯出備份檔」下載 JSON、「匯入備份檔」與 o 碼匯入共用取代/合併確認;lib 新增
   `serializeBackup`/`parseBackup`(kind 標記驗明正身)。chrome.storage.sync 方案經評估
   (非同步改造、配額、unpacked 擴充 ID 不固定)後由使用者決定不做。
4. **前端視覺收尾**(222294c):`.field select` 對齊 30px 輸入框樣式、稀有度 chips 加
   分組標題同款色點、「在 godfat 開啟清單」推右側與篩選分群。
