# Battle Cats Seed+ 多卡池比較

一個 Chrome / Edge（Manifest V3）瀏覽器擴充，讓你在 [godfat 種子追蹤頁](https://bc.godfat.org/) 一次勾選多個卡池（banner），於**單一頁面以「同抽數位置橫向對齊」**的方式比較。

由於所有卡池共用同一條種子序列，常見需求是「這一抽在哪個卡池最划算」。本工具把多個卡池並排，同一抽數位置（`1A`、`2A`…）排在同一列，一眼看出哪個卡池該位置是超激／保證 Uber，不必反覆切換卡池重新載入。

## 安裝（未封裝載入）

1. 開啟 `chrome://extensions`
2. 開啟右上角「開發人員模式」
3. 點「載入未封裝項目」，選擇本專案的 `extension/` 資料夾

## 使用

1. 在 godfat 的**種子頁**（網址含 `seed=...`）開啟頁面
2. 點擴充圖示 → 會自動帶入當前種子，並列出可選卡池（進行中者＋目前瀏覽中的卡池預設勾選）
3. 勾選想比較的卡池 → 按「載入比較」
4. 開啟比較頁：
   - 第一欄為抽數位置，每個卡池一欄，依位置橫向對齊
   - 稀有度以底色標示、目前位置有外框
   - 凍結首欄與表頭，可橫向捲動
   - 上方可勾選**隱藏／顯示**某卡池欄（不需重抓）、調整顯示抽數（**調小不重抓**：快取一池一份、就大不就小，只在調大時重抓該池）
   - **顯示 B 軌**勾選（預設關）：每卡池欄拆成 `A | B` 子欄，同一抽數的 A/B 軌並排對照（不需重抓）
   - **保證欄預設「原生」**：不帶 `force_guaranteed`，由 godfat 依該**場次**遊戲資料的必中旗標自動顯示（`guaranteed`→11 連、`step_up`→15 連、無旗標→空）；必中是場次屬性，同一卡池這期有、下期可能沒有。卡池列旁下拉可將個別卡池改為**強制 2/7/11/15**（`force_guaranteed` 模擬，localStorage 記住）；工具列「保證批次」為一次改全部的批次工具（全部回原生／全部強制 X），不存在持續生效的全域值。各格顯示「保證: …」與「保證(撞名): …」（撞名＝以重複狀態起手的變體），箭頭 `↪`＝落 B 軌前進、`↩`＝落回 A 軌
   - 每欄「在 godfat 開啟」可用相同種子在 godfat 開啟該單卡池原頁
5. **規劃路線**（挑目標貓 → 從 1A 求最短抽卡路線）：
   - 點表格格子開**選單**：加入該位置目標、加入該貓目標（所有出現擇一即可）、搜尋此貓（帶入 Find）、複製貓咪名稱；或用 Find 搜貓名後按「加入命中」批量加入
   - 按「規劃路線」→ 浮動面板列出 **Pareto 方案**（抽數 / GU / 白金券 / 傳說券 / 換軌數，互不被支配）；**GU＝確定連抽次數**：規劃器會在有保證型式的卡池自動考慮「單抽走位＋開確定連抽」的組合（含撞名起手），保證 Uber 與中間各抽都可收集目標，GU 起點格以雙框標示
   - 點方案 → 主表依序高亮路徑：步序號、換軌點（↹）、用特殊券格（虛線）、收集到目標的格（橙框）
   - 收不齊全部時，抽屜頂部列出未達目標與原因（互斥／軌道或時間不可達、卡池已結束、卡池已隱藏、或超出載入抽數），並給可行子集方案；隱藏中的卡池不會被排進路線
   - 路線方案面板預設浮動右上，**按住標題列可拖曳移動**，位置會記住（重載沿用）
   - 路線已考慮**卡池活動時間**：只在活動期間抽該卡池、時間單向前進（先抽較晚開始的卡池後，回不去屆時已結束的舊卡池）
   - 抽數與總券數相同時**白金優先於黑金**（白金券較便宜），僅保留白金方案
   - 換軌只計重複稀有（本版不含保證 Uber）；跨卡池才觸發、godfat 未標的重抽結果會標為「不確定」不計為收集
6. **擁有清單**（記錄你有的貓 → 標示新貨、批次目標）：
   - popup／view 工具列開「擁有清單」頁：按稀有度分組勾選、搜尋（含各型態名）、「只看未擁有」總覽
   - **匯入匯出走 godfat 原生 o 短碼**：在 godfat `/cats` 頁勾過的清單，貼網址即可匯入（取代或合併）；「在 godfat 開啟清單」可換取最新短碼、該網址兼作備份
   - view 勾「標示未擁有」→ 未擁有的貓在格子角落標小圓點（不動 godfat 底色）；點格子選單可直接「標記為已擁有」
   - 「在 godfat 開啟」連結會帶上清單（原站套 owned 底色）；本地改過清單會先向 godfat 換新短碼
   - 路線規劃「未擁有目標…」：列出表內未擁有的貓（按實際稀有度分組、預設全勾、可逐隻取捨）批量加入目標（面板顯示選取合計與規劃上限）

## 開發

```bash
npm install
npm test      # node:test + linkedom，對 godfat HTML fixture 做單元測試
```

- 純函式（`extension/lib/`）皆有單元測試：`godfat.js`（網址）、`parser.js`（解析）、`merge.js`（合併）、`concurrency.js`（並發）。
- 測試 fixture 與 godfat DOM 結構筆記見 `test/fixtures/`（含 `STRUCTURE.md`）。

## 重要實作細節

- **資料來源**：擴充以 `host_permissions` 直接抓取 godfat 各卡池頁面（避開 CORS），解析其 HTML。不自行重算種子（忠實沿用 godfat 計算）。
- **保證欄＝原生為主、強制為輔**：godfat 的「Guaranteed」表頭永遠存在，但欄位內容由**場次**必中旗標決定——godfat 的遊戲資料（`build/bc-*.yaml`）逐場次帶 `guaranteed: true`（11 連）或 `step_up: true`（15 連），無 `force_guaranteed` 參數時 godfat 即**自動填入**保證格（pick id 帶 `G` 字尾）；`force_guaranteed=2/7/11/15` 是在場次沒必中時**強制模擬**用的覆蓋。因此本工具預設不帶 force（呈現各場次真實必中），強制選項僅供模擬，不照卡池名猜測、不對任何卡池捏造保證；路線規劃的 GU 尺寸由解析結果的 G 格落點差推導。（註：2026-07-08 前的版本誤以為 godfat 從不自動填保證欄——當時實測對象是本來就沒必中旗標的白金/傳說票池——因而照卡池名捏造預設 11/15，已修正。）
- **稀有度 class**：`rare` / `supa` / `supa_fest` / `uber` / `uber_fest` / `exclusive`（`legend` 亦可能出現）。注意 class 是 godfat **全站固定的 score 區間**（6470/6970/9070/9470/9970 分界），不等於實際稀有度——機率非標準（rare≠6970）的卡池兩者會脫鉤，超國王祭（rare=6770）的 `(祭)` 帶甚至前段稀有、後段激稀有分裂。
- **稀有度過濾與撞名判定用「實際稀有度」**：以 godfat `/cats` 清單頁建「貓名→實際稀有度」對照表（每日抓一次、localStorage 快取，附貓 id），命中即為準；查無（離線、未來貓）時退回名單法近似（`(祭)` 格祭典卡池算該稀有度、非祭典降一級；白金/黑金票池必超激）。路線規劃的撞名重抽同樣以「實際為稀有」才觸發（godfat 的重抽僅發生在實際 Rare）。貓名搜尋與格子底色不受影響（底色忠實沿用 godfat）。
- **擁有清單以貓 id 為準、o 短碼為交換格式**：清單存 localStorage（`bcsp:owned`，清快取不清）。o 短碼是 godfat 伺服器端編碼——匯入靠解析 `/cats?o=…` 頁的 checked、匯出/換碼靠 `?t=<id>…` 網址讓 godfat 302 轉出短碼（全 793 隻的網址僅 ~4.7KB，實測可行）。`o` 對 godfat 是純顯示參數，不影響抽卡計算。

## 已知維護風險

godfat 改版可能使 HTML 結構改變、導致 `parser.js` 失效。`parser.js` 標有 `PARSER_VERSION`；修復時請更新 `test/fixtures/` 內的 fixture 與 `STRUCTURE.md`，並讓單元測試通過。

## 對 godfat 的禮貌性

godfat 是社群資源。本工具：抓取**並發上限 5**、localStorage 快取（一池一份就大不就小，縮小抽數不重抓；`/cats` 對照表每日至多抓一次）、不自動定時刷新。換碼請求只在開 godfat 連結且清單有改動、或手動匯出時發生（單次輕量請求）。請勿一次大量抓取。
